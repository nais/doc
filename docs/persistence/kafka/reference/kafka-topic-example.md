This is a complete example of an `Topic` resource, commonly known as the `topic.yaml` file.

For an in-depth explanation of each field, head over to the [reference documentation](./kafka-topic-spec.md).
``` yaml
apiVersion: kafka.nais.io/v1
kind: Topic
metadata:
  creationTimestamp: null
  labels:
    team: myteam
  name: mytopic
  namespace: myteam
spec:
  acl:
  - access: read
    application: consumer
    team: otherteam
  - access: write
    application: producer
    team: myteam
  - access: readwrite
    application: processor
    team: myteam
  config:
    cleanupPolicy: delete
    maxMessageBytes: 1048588
    minimumInSyncReplicas: 2
    partitions: 1
    replication: 3
    retentionBytes: -1
    retentionHours: 168
    segmentHours: 168
  pool: nav-integration-test
```
